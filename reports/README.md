# Test Reports Directory

This directory contains Robot Framework test execution reports generated after each test run.

## Report Types

### 1. **output.xml** (Machine-readable)
- Format: XML
- Contains: Complete test execution data
- Used by: CI/CD tools, report generators, parsers
- Size: Medium (100-500 KB)

### 2. **log.html** (Detailed Log)
- Format: HTML
- Contains: Step-by-step test execution details
- Used by: Debugging, detailed analysis
- Features:
  - Expandable test steps
  - Keyword execution times
  - Arguments and return values
  - Screenshots (if any)
  - Error messages with stack traces

### 3. **report.html** (Summary Report)
- Format: HTML
- Contains: High-level test statistics
- Used by: Quick overview, management reporting
- Features:
  - Total tests: passed/failed
  - Test execution times
  - Tags statistics
  - Suite-level results
  - Trend graphs (if configured)

### 4. **screenshots/** (Visual Evidence)
- Format: PNG images
- Contains: Browser screenshots on failure
- Used by: Visual debugging
- Naming: `selenium-screenshot-{timestamp}.png`

### 5. **xunit.xml** (JUnit Format)
- Format: XML (JUnit compatible)
- Contains: Test results in JUnit format
- Used by: Jenkins, CI/CD dashboards
- Generate: `--xunit xunit.xml` flag

## Viewing Reports

### Local
```bash
# Windows
start reports/report.html

# Linux
xdg-open reports/report.html

# Mac
open reports/report.html
```

### Jenkins
Reports are automatically archived:
- Console Output: Build → Console Output
- Test Results: Build → Test Result
- Artifacts: Build → Build Artifacts → reports/

## Report Generation

Reports are automatically generated by Robot Framework:
```bash
python3 -m robot --outputdir reports robot-tests/
```

Custom report options:
```bash
# With timestamp
python3 -m robot --outputdir reports --timestampoutputs robot-tests/

# With custom names
python3 -m robot --outputdir reports \
    --output my_output.xml \
    --log my_log.html \
    --report my_report.html \
    robot-tests/

# Combine multiple runs
rebot --outputdir reports output1.xml output2.xml output3.xml
```

## Report Retention

**Git:** Reports are ignored (see `.gitignore`)
**Jenkins:** Last 10 builds retained (configurable)
**Archive:** Important reports should be backed up externally

## MongoDB Integration

Failed tests are also sent to MongoDB Atlas:
- Collection: `test_failures`
- Dashboard: http://localhost:5173
- Query: Real-time failure analysis

## Troubleshooting

**No reports generated?**
- Check: Test execution completed successfully
- Check: `--outputdir` path is writable
- Check: Disk space available

**Reports not in Jenkins?**
- Configure: Post-build Actions → Publish Robot Framework test results
- Path: `reports/output.xml`

**Can't open HTML reports?**
- Security: Some browsers block local HTML + JS
- Solution: Use `file://` protocol or local web server
